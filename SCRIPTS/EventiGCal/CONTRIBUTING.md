## 📝 Come Contribuire

### 1. **Segnalare un Bug**
Se trovi un bug, apri una **Issue** su GitHub con:
- Una descrizione chiara del problema.
- Passaggi per riprodurlo.
- Eventuali screenshot o log di errore.

### 2. **Suggerire una Nuova Funzionalità**
Se hai un’idea per migliorare lo script:
- Apri una **Issue** con una descrizione dettagliata.
- Spiega perché ritieni che la funzionalità sia utile.

### 3. **Inviare una Pull Request**
Se vuoi contribuire direttamente con del codice:
1. **Forka** il repository.
2. Crea un **branch** con un nome descrittivo (es. `fix/bug-link` o `feat/nuova-funzione`).
3. Fai le tue modifiche, assicurandoti che:
   - Il codice sia **pulito** e **commentato**.
   - Non ci siano errori di sintassi o logica.
4. Apri una **Pull Request** verso il branch `main` del repository originale.
5. Attendi la revisione e rispondi a eventuali feedback.

---

## 📌 Standard di Codice
- Usa **Python 3.x**.
- Segui lo stile di codice esistente (es. nomi di variabili, indentazione).
- Aggiungi **commenti** dove necessario per spiegare la logica.
- Se aggiungi nuove funzionalità, aggiorna anche il **README.md** e la documentazione.

Sii **rispettoso** e **costruttivo** nelle discussioni. Questo progetto mira a essere un ambiente **inclusivo** e **collaborativo**.
